# Contributing to SubLink Relay

感谢你对 SubLink Relay 的关注！我们非常欢迎任何形式的贡献，包括提交 Issue、改进文档或提交代码。

## 提交 Issue

在提交 Issue 之前，请：
1. 搜索现有的 Issue，看看是否已经有人提出过类似的问题。
2. 如果是 Bug 报告，请提供详细的复现步骤、环境信息（Node.js 版本、操作系统等）以及错误日志。
3. 如果是功能建议，请详细描述使用场景和预期的功能行为。

## 本地开发

1. Fork 本仓库并克隆到本地。
2. 安装依赖：`npm install`
3. 启动开发服务器：`npm run dev`
4. 确保代码通过 lint 检查：`npm run lint`

## 提交 Pull Request

1. 请确保你的代码风格与项目保持一致。
2. 提交 PR 前，请运行 `npm run lint` 和 `npm run format` 确保代码格式正确。
3. 如果引入了新功能，建议补充相应的测试用例（如有）。
4. PR 标题应简洁明了，描述本次修改的主要内容。

## 代码规范

本项目使用 ESLint 和 Prettier 进行代码规范检查。
- **Lint**: `npm run lint`
- **Format**: `npm run format`

感谢你的贡献！
